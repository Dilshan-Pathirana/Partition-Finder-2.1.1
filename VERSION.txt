CHANGES FROM ORIGINAL PARTITIONFINDER 2.1.1
--------------------------------------------
This version is a complete Python 3 port of the original PartitionFinder 2.1.1

Major Changes:
- Migrated entire codebase from Python 2.7 to Python 3.8+
- Updated 30+ Python 2 specific syntax patterns
- Replaced deprecated modules and functions
- Fixed all bytes/string encoding issues
- Updated dependency versions to latest stable releases
- Removed Python 2 backward compatibility code

Technical Fixes:
- Replaced print statements with print() functions
- Changed xrange() to range()
- Changed dict.iteritems() to dict.items()
- Changed dict.itervalues() to dict.values()
- Removed itertools.izip (use built-in zip)
- Updated exception syntax (except X, e → except X as e)
- Replaced imp module with importlib.util
- Replaced cPickle with pickle
- Replaced cStringIO with io module
- Updated time.clock() to time.perf_counter()
- Fixed numpy.fromstring() to array comprehension
- Fixed numpy.tostring() to numpy.tobytes()
- Updated scipy.misc.comb to scipy.special.comb
- Fixed scikit-learn KMeans (removed n_jobs parameter)
- Changed file open modes ('rU' → 'r', 'wb' → 'w' for text)
- Fixed integer division (/ → // where appropriate)
- Added bytes/string decoding in model utilities

Testing Status:
- ✅ DNA/Nucleotide analysis: PASSED
- ✅ Protein/Amino acid analysis: PASSED  
- ✅ Morphological analysis: VERIFIED

KNOWN ISSUES
------------
- K-means clustering can be slow on large datasets
- Morphological analyses require --raxml flag
- Windows users must use 'py -3.12' command instead of 'python'
